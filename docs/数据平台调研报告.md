## 二、各平台详细分析

### 1. FastMoss（主力平台）⭐ 推荐作为主数据源

**官网：** https://www.fastmoss.com
**OpenAPI 开发者门户：** https://developers.fastmoss.com/zh/
**OpenAPI 定价页：** https://developers.fastmoss.com/zh/pricing.html
**原始接口数据（已抓取）：** [`scripts/fastmoss_endpoints.json`](./scripts/fastmoss_endpoints.json)

#### API 能力（按接口计费，共 22 个接口）

FastMoss OpenAPI 已开放按接口购买，接口列表和单价已从定价页 API `/developer/payment/endpointsList` 抓取（2026-04-07）：

**商品数据（Product，$0.01/次）**
- `/product/v1/search` — 关键词/筛选搜索 TikTok Shop 商品
- `/product/v1/goodSalesList` — 商品历史销量
- `/product/v1/videoList` — 商品关联视频

- `/product/v1/reviewList` — 商品评论
- `/product/v1/rank/topSelling` — 热销商品榜
- `/product/v1/rank/mostPromoted` — 最多推广商品榜
- `/product/v1/rank/fullyManaged` — 全托管商品榜
- `/product/v1/rank/newListed` — 新上架商品榜

**达人数据（Creator，$0.02/次）**
- `/creator/v1/search` — 达人搜索
- `/creator/v1/productList` — 达人带货商品列表

#### 数据实时性

- **更新频率：秒级更新**，官方宣称数据更新达到秒级
- 实时捕捉 TikTok 市场变化
- 这是所有第三方平台中数据更新最快的

#### OpenAPI 定价（按接口调用量计费）

FastMoss OpenAPI 采用**按接口 + 按次计费**模式，没有传统月费套餐。定价页显示货币符号会随语言切换 ¥/$，数字相同（以下按美元列示，中文站展示为 ¥ 同值；以实际订单结算为准）：

| 接口类别 | 单价（每次请求） | 每 1,000 次成本 | 最小购买增量 |
| ---- | -------- | ---------- | ------ |
| 商品 Search / 排行榜 / 销量 / 视频 / 评论 | $0.01 | $10 | 1,000 次 |
| 小店 Search / 商品列表 / 视频 / 直播 | $0.01 | $10 | 1,000 次 |
| 视频搜索 / 直播搜索 | $0.01 | $10 | 1,000 次 |
| 达人所有接口（含商品/小店的关联达人） | $0.02 | $20 | 1,000 次 |

**额外方案：**

| 方案 | 价格 | 额度 | 说明 |
| -- | -- | -- | -- |
| 免费试用 | $0 | **500 次总量/月** | 标准数据延迟，邮件支持 |

**MVP 粗算（按 方案 B 推荐用量）：**

- 商品 search + 排行榜 + 销量：合计 ~20,000 次/月 × $0.01 = **$200**
- 达人 search + 带货列表：~5,000 次/月 × $0.02 = **$100**
- 小店/视频/直播辅助：~5,000 次/月 × $0.01 = **$50**
- **月度估算 ≈ $350（约 ¥2,500）**，年付按官方提示最高可省 20%

#### 评估结论

- **优势：** 数据量最大、更新最快、**OpenAPI 定价公开透明、按接口按次计费**，无需强制订阅套餐；免费 500 次可立即验证
- **劣势：** 达人类接口单价翻倍（$0.02/次）；1,000 次最小增量意味着就算只用几次也得买 1,000；高频轮询成本会快速累加
- **适合度：** ★★★★★ — 作为我们 AI 系统的主数据源最合适，且可先用免费额度联调 MVP

#### 评估结论

- **优势：** 团队源自 TikTok 全球电商团队（数据理解深），500天历史数据，达人四象限分析独特
- **劣势：** API 仅限企业版，价格门槛高；数据更新 15 分钟（vs FastMoss 秒级）

### 4. TikTok 官方 API

#### 4.1 TikTok Shop Partner API v2（卖家 API）

**文档入口：** https://partner.tiktokshop.com/docv2/page/seller-api-overview

**API 能力：**
- **Products API**：商品管理（创建、更新、查询商品信息）
- **Orders API**：订单管理（查询订单、更新状态）
- **Logistics API**：物流管理
- **Finance API**：财务数据
- 按 OAuth 认证，需签名请求

**特点：**
- 仅能访问**自己店铺**的数据，无法看到竞品/市场数据
- 需要在 Partner Center 注册应用，申请对应 scope
- 各地区端点可用性可能不同

#### 4.2 TikTok API for Business（广告 API）

**文档入口：** https://business-api.tiktok.com/portal

**API 能力：**
- 广告投放管理
- 广告数据报告
- Events API（服务端转化追踪）
- 受众管理

**特点：**
- 可程序化管理广告（这对我们的投流优化模块很重要）
- 提供广告效果数据报告

#### 4.3 TikTok Research API / Trends

**API 能力：**
- 匿名化、聚合的公开数据
- 用户行为和趋势研究
- **仅对获批的研究者开放**，不适用于商业用途

#### 评估结论

- **优势：** 数据最准确（一手数据），免费，可管理自己店铺和广告
- **劣势：** 无竞品/市场数据，Research API 申请门槛高
- **适合度：** ★★★★☆ — 必须接入（自己店铺运营数据 + 广告管理），但无法替代第三方平台的市场数据

### 9. 1688 开放平台（供应链数据源）

**官网：** https://www.1688.com
**开放平台：** https://open.1688.com
**API 文档：** https://open.1688.com/api/overview.htm

#### 平台定位

1688 是阿里巴巴旗下国内最大的 B2B 批发采购平台，连接国内工厂/供应商与采购商。对跨境电商而言，1688 是**货源端**的核心数据源——与前面 8 个平台关注"销售端"（TikTok 数据）不同，1688 提供的是**成本端**数据，直接影响 ROI 计算中的商品成本项。

#### ⚠️ 重要修正（基于实际文档核实）

经实际查看 1688 开放平台接口文档后发现：**1688 开放平台的大部分 API 是面向 ISV 服务商 + 1688 卖家设计的，而非面向买家/采购方**。例如：

- `alibaba.product.get` 文档页明确注明："**由商品ID获取商品详细信息，请注意：只能查询自己所有的产品**"——即只能查询当前账号作为卖家已发布的商品，不能查询其他供应商的商品。
- 产品类目下的大部分接口（增量修改、删除、修改库存等）都属于"店铺管理"场景，对买家没有意义。

对我们这种"跨境电商买家想查 1688 供应商商品"的场景，**1688 开放平台并没有公开可用的"全站商品搜索/比价" API**。

#### ⚠️ 进一步核实：大企业采购解决方案2.0（BuyerPurchase）也不适合我们

经核实 https://open.1688.com/solution/detail?key=1593395762875&category=BuyerPurchase （大企业采购解决方案2.0，版本13.8，更新于2024-7-11），该方案明确：

- **目标客户**："本方案主要针对**购买了大企业采购 SaaS 产品的企业客户**，需要将企业内部使用的供应商管理系统、物料管理系统、ERP 系统以及 SRM 等系统与 1688 平台进行系统对接"
- **接入前提**：必须先"订购《大企业采购解决方案》"（付费 SaaS），非大企业采购客户需手动申请"采购服务商"角色资质
- **完整 API 清单覆盖**：账号同步、组织架构、物料主数据、供应商主数据、请购、询价、比价、内部商城、合同、订单、地址、物流、履约、招投标、竞价单、寻源消息——**全部是企业内部采购流程数字化 API**
- **关键缺失**：整个 API 清单中**没有"1688 全站商品搜索"接口**。`mallgoods.search` 类接口搜的是买家**自己内部商城目录**，不是 1688 公开商品池
- **工作流不匹配**：所有流程都假设"供应商已知"——询价→比价→下单给已建档供应商。我们的场景是"从 1688 海量商品池中发现新货源"，这条路径根本走不通

**结论**：大企业采购解决方案是给已有固定供应商的大型甲方做内部采购数字化的 SaaS 对接方案，**不是为"跨境卖家选品+比价"设计的**，对我们无用。

#### 真正可用的 API 模块（买家/采购侧）

| 模块 | 说明 | 接入方式 |
|------|------|---------|
| **大企业采购方案** | 1688 专为企业采购方设计的完整 API 套件，覆盖商品查询、询价、下单、支付、物流、对账、发票 | 需通过 1688 商务对接，填写企业资质和采购规模说明，审核周期以周为单位 |
| **跨境选品搜索** `alibaba.cross.company.product.new.search` | 跨境场景专供的选品搜索接口（是否对我们这种规模开放需实测） | 可能需购买"跨境选品解决方案"权限包或走商务合作 |
| **交易类 API** `alibaba.createOrder.preview`、`alibaba.trade.fastCreateOrder` | 采购下单预览、快速下单（通常作为大企业采购方案的一部分） | 随大企业采购方案一起申请 |

> ⚠️ 这意味着 1688 API 的接入**不是"注册开发者就能自助申请"的简单流程**，而是需要走 1688 商务对接的重型流程。我最初的调研结论过于乐观，特此更正。

#### 接入条件（⚠️ 硬性门槛）

| 要求 | 说明 |
|------|------|

#### 对跨境电商 ROI 计算的价值

1688 数据是我们 ROI 公式中**成本端**的核心输入：

```
ROI = (销售额 - 总成本) / 总成本

总成本 = 商品成本 + 国内运费 + 跨境运费 + 广告费 + 达人佣金 + 平台抽成
          ↑            ↑
       1688 采购价   1688 供应商发货
```

通过 1688 API 可以实现：
- **智能选品**：搜索 TikTok 热卖品对应的 1688 货源，自动计算采购成本和潜在利润
- **比价引擎**：同一商品多供应商自动比价，筛选最优成本
- **库存监控**：实时追踪供应商库存变化，避免断货
- **一键代发**：通过交易 API 实现采购下单自动化（Phase 4）

#### 评估结论

- **优势：** 数据直接关联成本端，阿里官方、权威性最高
- **劣势：**
  1. **常规产品 API 不可用**——`alibaba.product.get` 等只能查自己店铺的商品，不是买家视角；
  2. **大企业采购解决方案2.0 也不适合**——该方案面向已有固定供应商的大型企业做内部采购数字化，整个 API 清单里没有"1688 全站商品搜索"接口，工作流假设供应商已知，与"跨境卖家选品/比价"场景完全不匹配；
  3. 前置成本：诚信通 AI 版 ¥9,988/年 + 大企业采购 SaaS 订购费；
  4. 跨境选品接口 `alibaba.cross.company.product.new.search` 的开放范围未知，可能同样需要购买专项权限包，且无公开申请入口。
- **适合度（官方）：** ★☆☆☆☆ — **1688 开放平台对我们的场景没有可直接使用的买家侧公开 API**。

#### 9.Y MVP 选型决策：双源容灾 + 本地缓存

```
          业务层（选品/成本/ROI 引擎）
                    │
          ┌─────────▼──────────┐
          │  1688 数据抽象层     │  统一接口: search/detail/imageSearch/shop

             │          │
        ┌────▼───┐ ┌────▼────┐
        │ 万邦(主)│ │Pangolin │  failover ≤15s
        └────┬───┘ └────┬────┘
             └──────┬───┘
                    ▼
             本地 Redis 缓存（24h）
```

- **主源**：万邦 Onebound 按量付费 ¥0.09/次
- **备源**：Pangolin 或 Elimapi 按量付费（低频，仅主源失败时启用）
- **缓存**：本地 24h 缓存，重复查询不产生 API 费用
- **SLA 目标**：可用性 ≥99%（与 Phase 0 MVP 验收标准一致）
- **合规边界**：**仅用于查询/选品/成本估算，不接入任何代理 API 的下单/支付接口**；交易环节维持人工

- **官方 1688 开放平台**：★☆☆☆☆ 不适用

- **成本控制**：MVP 阶段预估调用量 3000-8000 次/月，按量付费总预算约 ¥400-1000/月（主源 ¥300-800 + 备源 ¥100-200），远低于官方路径的 ¥9,988/年+SaaS 订购费

## 五、关键风险与注意事项

### 1. 反爬与合规风险
- TikTok 反爬非常严格，直接爬取平台数据风险极高
- 建议**仅通过官方 API 和正规第三方平台 API 获取数据**，不做任何直接爬取
- FastMoss/Kalodata 的数据采集合规性由它们自己负责，我们作为 API 用户不承担此风险

### 2. 数据准确性
- **FastMoss/Kalodata 的销售数据是估算值**，非精确数据（TikTok 不公开精确销售数据给第三方）
- Kalodata 声称数据来源于 TikTok 官方 API，但实际上 TikTok 没有公开的竞品数据 API
- 建议使用时做**交叉验证**：对比多平台数据，取共识值

### 3. 日本市场覆盖
- FastMoss 明确覆盖 15+ 国家/地区（含日本）
- Kalodata 也覆盖日本
- EchoTik 的日本市场覆盖程度需进一步确认
- 日本市场数据可能比美国/东南亚市场少，需实际测试数据丰富度

### 4. API 稳定性与限流
- 第三方 API 可能有调用频率限制和不定期维护
- 建议架构设计中加入**数据缓存层**，避免过度依赖实时 API 调用
- 关键数据定期同步到本地数据库
