## 第一条 项目概述

1.1 项目名称：TikTok ROI 智能运营平台（MVP 阶段）。

1.2 项目目标：面向跨境电商（日本市场）开发 AI 驱动的运营效率提升工具，以 ROI 为核心指标，构建**"ROI 定价决策工作台"**：让运营在选品/上品阶段输入货源成本和产品信息后，系统输出该商品的定价建议、ROI 测算、竞品对标数据，并将决策结果存档复用。

1.3 交付形式：单一 **Web 应用**（含前端、后端 API、数据接入模块、LLM 集成）；源代码、部署文档、使用说明一并交付。MVP 阶段**不含**浏览器插件、移动端 App、原生桌面应用。

## 第二条 工期安排

2.1 总工期：自合同生效之日起共 30 个工作日（含开发、联调、测试与缓冲时间）。

2.2 工期构成：Phase 0 数据源接入与基础架构（约 6 天）；Phase 1 MVP 核心功能 F1–F7 开发（约 18 天）；联调、测试与部署（约 4 天）；缓冲与修复（约 2 天）。

2.3 若因甲方原因（如需求变更、资源未及时提供、数据平台商务流程延误等）导致工期延后，则工期相应顺延，顺延天数经双方书面确认。

## 第六条 MVP 验收标准

6.1 功能清单 F1–F7（乙方须全部交付并通过验收；详细产品设计以《MVP产品设计.md》为准）：

| 编号 | 模块 | 功能点 | 交付标准 |
|---|---|---|---|
| **F1** | 运费计算器 | 根据商品克重、货品类型（普货/特货/敏货）自动计算跨境直邮运费 | 支持包邮/不包邮两种模式；支持多打包组合（1 件/5 件/10 件装）对比；运费表配置化可在管理后台更新；与 TikTok 官方日本直邮运费规则一致 |
| **F2** | ROI 计算器 & 盈亏模拟 | 输入四项成本（货源/物流/投流/达人佣金）双向计算：给定售价算 ROI，或给定目标 ROI 反推最低售价 | 输出零售价/折扣价/秒杀价三档建议 + 盈亏平衡点；支持多 SKU 差异化定价；ROI 保本线默认 2.5 可配置 |
| **F3** | 竞品价格带分析 | 通过 FastMoss OpenAPI 获取同品类商品价格区间、热销榜、历史销量 | 接入 `/product/v1/search`、`/rank/topSelling`、`/goodSalesList` 三个核心接口；支持日本市场；展示价格分布图、Top 20 对标品、销量/价格散点图；结果按"关键词+日期"缓存 24 小时 |
| **F4** | LLM 智能定价建议 | 基于 F2 成本结构 + F3 竞品数据 + 产品描述，调用 Claude Sonnet 4.6 输出结构化建议 | 输出三档建议价 + 理由 + 市场定位（低价引流/中位跟随/高价差异化）+ 风险提示；通过阿里云 Proxy 转发；开启 Prompt Caching 控成本；历史建议保留可对比 |
| **F5** | 商品工作台 | 商品 CRUD + 决策存档 + 历史复盘 | 每个商品保存基础信息、成本结构、竞品参考池、历史定价建议、最终上架价；支持列表搜索/筛选；同一商品多次生成建议可对比；**MVP 核心数据资产** |
| **F6** | 店铺自测 Dashboard | 通过 TikTok Shop Partner API v2 拉取自店当日订单、GMV、实测 ROI | 至少含 3 张指标卡（GMV / 订单数 / 店铺总 ROI）+ 最近 10 条定价决策记录；支持 AI 建议价 vs 实跑 ROI 偏差对比；OAuth 审批延期时提供订单 CSV 手动导入兜底 |
| **F7** | 1688 图搜反查货源 | 通过万邦 Onebound `item_search_img` 接口以图搜同款返回 1688 货源价 | 作为 F2 货源成本的外部参考值；Pangolin/Elimapi 仅预留数据抽象层接口，备源具体实现不在 MVP 范围 |

6.1.2 **MVP 明确不含的功能**（写入合同以防范围蔓延，后续阶段另行签约）：AI 达人评估系统、投流优化助手、内容/视频/标题 AI 生成、浏览器插件（自动填字段）、订单智能分拣、活动到期提醒与续期、官方活动推荐、多店铺/多租户。

6.1.3 **用户体系**：MVP 提供基础账号登录与权限控制，支持运营（主用户）/ 老板（只读 Dashboard）/ 管理员 三种角色；MVP 阶段面向单一店铺。

6.2 性能指标：

| 指标                 | 目标值                                       |
| ------------------ | ----------------------------------------- |
| Web 界面响应时间（常规页面加载） | ≤ 2 秒                                     |
| 竞品数据 API 查询响应      | ≤ 3 秒（含缓存命中）                              |
| 1688 代理 API 查询响应   | ≤ 3 秒（含缓存命中）                              |
| 系统可用性（MVP 阶段）      | ≥ 99%（非计划性停机除外）                           |
| 并发支持               | ≥ 10 个运营同时使用无明显卡顿                         |
| 数据准确性              | 运费计算与 TikTok 官方规则 100% 一致；ROI 计算公式经甲方书面确认 |
| 缺陷率（验收时）           | P0/P1 级缺陷为 0；P2 级缺陷 ≤ 5 个                 |

6.3 验收流程：乙方完成开发并自测通过后，向甲方提交《MVP 交付说明书》；甲方应在 5 个工作日内完成验收测试。如甲方在期限内未提出书面异议，视为验收通过。

6.4 缺陷等级定义：P0（阻塞核心业务）、P1（核心功能异常）、P2（非核心功能缺陷或体验问题）、P3（轻微问题）。

## 第七条 需求变更应对

7.1 本合同签订后，如甲方需在 MVP 范围外增加、修改或删减需求，须以书面形式（含邮件、企业微信等电子方式）提出变更申请。

7.2 变更评估：乙方应在收到变更申请后 2 个工作日内，向甲方提交《需求变更影响评估》，内容包括工作量评估、对工期的影响、对费用的影响。

7.3 变更分级处理：

- **轻微变更**：单项变更工作量 ≤ 0.5 人天，且总累计 ≤ 2 人天，乙方免费实施，不影响工期与费用。
- **中等变更**：单项变更工作量 > 0.5 人天 且 ≤ 3 人天，按 ¥4,000/天 追加费用，工期相应顺延。
- **重大变更**：单项变更工作量 > 3 人天，或涉及架构调整、核心流程重构，需双方另行签订《变更协议》，明确新增费用、工期和交付范围。

7.4 未经书面确认的变更，乙方有权不予实施；因甲方口头要求擅自实施导致的争议，由甲方承担。

7.5 变更不得影响已交付且验收通过的功能模块稳定性；如变更导致返工，返工工作量计入变更工作量。
