- 店铺面向日本市场，数据源（TikTok / FastMoss）API 端点多在新加坡/日本，东京区访问延迟 < 50ms

- 新增 **Claude Runner**（EC2 t3.large）作为日常开发工作机 + 自动化脚本执行环境

## 七、风险项与成本波动因素

| 风险 | 影响 | 应对措施 |
|---|---|---|
| FastMoss 日本达人覆盖度 < 1,000 | F7 降级为"列 Top 50，不做交互式筛选" | Phase 0 先做 quota 覆盖度预测试，合同已写降级条款 |
| FastMoss 调用量超出基线 1200 链接/月 | FastMoss 成本线性上升 | 30% buffer 已含；超出 buffer 后按增量补充，由 李远哲 走付款 |
| Onebound `item_weight` 高频为空 | F6 → F1 → F2 不闭环 | 提示运营手填重量，合同兜底 |
| 图搜跨类目召回 | F6 货源不准 | 类目交叉验证层，不匹配标记低置信度 |
| TikTok 运费价卡变动 | F1 数据过时 | 价卡 JSON 配置化 + 运营手动触发刷新脚本 |
| Claude API 用量超预期 | LLM 成本上升 | Prompt Caching + Batch API 降本；合同上限 ¥1,000 |
| AWS 东京区新用户优惠到期 | 第二年续费涨价 | Phase 1 阶段按 2 个月预算锁定，不涉及续费 |
| 需求变更超出原始范围 | 开发工时增加 | 合同 §7 变更分级：≤ 0.5 天免费，> 3 天走变更协议 |

## 八、省钱路径（可选）

1. **走通 1688 官方开放平台 API** → 省 ¥2,000 Onebound 充值
2. **FastMoss 实际用量低于 6,500 次/月** → 省 ~¥300-500（由 API 按量结算自动节省）
3. **Claude Prompt Caching 命中率高** → 省 ~¥500（自动节省）
4. **AWS 使用 Reserved Instance**（若 Phase 2 续跑同架构）→ 省 20-30%
