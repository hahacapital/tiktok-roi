## 四、基础设施费用（阿里云）

### 架构说明

```
                    ┌──────────┐
  用户 ──── ALB ────┤ 应用服务器  │──── 云数据库 MySQL
                    │ 4C8G ECS │
                    └────┬─────┘
                         │
                  ┌──────┴──────┐
                  │ Proxy Server │──── Claude API
                  │  2C4G ECS   │     (海外 LLM)
                  └─────────────┘
```

- **应用服务器（4C8G）**：运行 Web 应用、后端 API、数据缓存
- **Proxy Server（2C4G）**：转发 LLM 请求到 Claude API（解决国内访问海外 API 的网络问题）
- **云数据库 MySQL（2C4G）**：存储商品、竞品、运费、投流等业务数据
- **ALB 负载均衡**：流量入口，HTTPS 终结，后续可扩展多实例

### 费用明细

| 资源 | 规格 | 计费方式 | 月费 | MVP 期间费用（2个月） | 说明 |
|------|------|----------|------|---------------------|------|
| 应用服务器 ECS | 4核8G, 5M带宽, 40G ESSD | 包年优惠 | ¥75/月（898/年） | ¥150 | u2a 通用算力型，新用户价 |
| Proxy Server ECS | 2核4G, 5M带宽, 80G ESSD | 包年优惠 | ¥17/月（199/年） | ¥34 | u1 实例，部署海外区域（如香港/新加坡）用于转发 LLM 请求 |

| 域名 | .com | 年付 | — | ¥60 | 如 tiktok-roi.com 等 |

## 五、LLM API 费用（Claude，通过 Proxy 转发）

### 使用场景

| 场景 | 推荐模型 | 频率 | 说明 |
|------|----------|------|------|
| 智能定价分析 | Sonnet 4.6 | 每次定价调用 | 综合竞品数据 + 成本结构，输出定价建议 |

| 模型 | 单价（输入/输出） | 预估月用量 | 月费 | 2个月费用 |

| Sonnet 4.6 | $3/$15 per 1M tokens | 2M input + 0.5M output | $13.5 ≈ ¥98 | ¥196 |
| Haiku 4.5 | $1/$5 per 1M tokens | 5M input + 1M output | $10 ≈ ¥73 | ¥146 |
| **小计** | | | | **¥342** |

> 注：以上为保守估算，实际用量取决于日均调用次数。

## 八、风险项与成本波动因素

| 风险                  | 影响          | 应对措施                             |
| ------------------- | ----------- | -------------------------------- |
| FastMoss 调用量超出预估（3.5 万次/月） | 数据平台成本线性上升 | 监控调用量，命中高频接口加 Redis 缓存；必要时切换 Tabcut/EchoTik |
| TikTok 官方 API 审批延迟  | 自店数据无法接入    | 先用手动导出数据过渡                       |
| 日本市场第三方数据偏少         | 定价建议准确度下降   | 增加人工校验环节，积累自有数据                  |
| 需求变更超出原始范围          | 开发工时增加      | 变更超过 20% 需重新评估报价                 |
| Claude API 用量超预期    | LLM 成本上升    | 启用 Prompt Caching + Batch API 降本 |
| Proxy 网络不稳定         | LLM 调用延迟或失败 | 增加重试机制 + 备用 Proxy 节点             |
| 阿里云优惠到期续费涨价         | 基础设施年化成本上升  | 提前锁定多年优惠，或评估迁移方案                 |
