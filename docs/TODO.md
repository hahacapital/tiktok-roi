## Phase 1（MVP）：ROI 定价决策工作台 · F1–F7（第 2-5 周）

> **MVP 产品形态：单一 Web 应用"ROI 定价决策工作台"**。完整设计见 [`MVP产品设计.md`](./MVP产品设计.md)。
> 严格按 F1–F7 执行，任何达人评估 / 投流 / 内容生成 / 浏览器插件需求一律拒绝，归到 Phase 2+。

### F1. 运费计算器（2 天）

- [ ] **提供 TikTok 商家 Portal 运费表** — @孙栩尧
  - 普/特/敏货费率、各重量段单价、基础包裹运费（当前 475 日元）
  - 包邮 vs 不包邮规则
  - 输出成 CSV 或截图发给 @张翼翔

- [ ] **F1-dev：运费计算模块** — @张翼翔
  - 输入：克重 / 货品类型（普/特/敏）/ 是否包邮 / 目的国（MVP 固定日本）
  - 输出：单件物流成本、多打包组合（1/5/10 装）对比
  - 运费表配置化（"运费与规则"页后台维护，不硬编码）

### F2. ROI 计算器 & 盈亏模拟（3 天）

- [ ] **确认 ROI 公式和变量口径** — @孙栩尧 @张翼翔
  - 成本结构：货源成本 + 物流（F1） + 投流 + 达人佣金
  - ROI = GMV / 总成本；保本阈值 2.5
  - 确认达人佣金默认区间（10%-30%）

- [ ] **F2-dev：ROI 双向计算器** — @张翼翔
  - 正向：给定售价 → 算 ROI
  - 反向：给定目标 ROI（默认 2.5）→ 反推最低售价
  - 输出：零售价 / 折扣价 / 秒杀价 三档 + 盈亏平衡点
  - 多 SKU 分组计算（单装/5 装/10 装）

- [ ] **成本数据录入模板** — @孙栩尧
  - 对接赵总的成本提供方式（Excel？口头？）
  - 标准化录入字段

### F3. 竞品价格带分析（3 天）

- [ ] **F3-dev：FastMoss 竞品接口对接** — @张翼翔
  - `/product/v1/search`（关键词搜索）

  - `/product/v1/goodSalesList`（历史销量）
  - 结果按"关键词+日期"缓存 24h（避免重复计费）

- [ ] **F3-ui：价格带可视化** — @张翼翔

  - "加入参考池"按钮 → 关联到某个商品卡

### F4. LLM 智能定价建议（3 天）

- [ ] **F4-prompt：设计定价建议 Prompt** — @张翼翔 @孙栩尧
  - 输入：F2 成本结构 + F3 竞品摘要 + 产品描述
  - 输出结构化 JSON：零售/折扣/秒杀三档 + 理由 + 市场定位 + 风险提示
  - 孙栩尧提供 2-3 个历史定价案例作为 few-shot 示例

- [ ] **F4-dev：LLM 集成 + 建议存档** — @张翼翔

  - 开启 Prompt Caching 控成本
  - 同一商品可多次生成，历史建议可对比

### F5. 商品工作台（3 天）⭐ MVP 灵魂

- [ ] **F5-ui：工作台主界面** — @张翼翔

### MVP 量化验收（Phase 1 结束时由 @孙栩尧 @赵总 评估）

- [ ] **量化指标验收** — @孙栩尧 @赵总
  - 定价决策耗时：目标 ≤ 20 分钟/品（当前约 120 分钟）
  - ROI 预测偏差率：AI 建议价 vs 实跑 ROI，偏差 ≤ 15%
  - 决策存档命中率：≥ 50% 新品能复用已存档参数
  - 不达标则继续打磨 Prompt / 数据源 / 权重
